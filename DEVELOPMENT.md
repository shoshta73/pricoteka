# Development

This document covers the day-to-day development workflow for Pricoteka.

## Prerequisites

- Node.js `24.15.0`
- pnpm `11.5.0`

Install dependencies from the repository root:

```sh
pnpm install
```

## Project Structure

Pricoteka is a pnpm workspace with four main packages:

- `apps/frontend/` - React frontend, published in the workspace as `@pricoteka/app`
- `apps/api/` - Hono API server, published in the workspace as `@pricoteka/api`
- `website/` - marketing website, published in the workspace as `@pricoteka/website`
- `packages/core/` - shared schemas and exports, published in the workspace as `@pricoteka/core`
- `packages/ui-core/` - shared shadcn UI primitives and theme CSS, published in the workspace as `@pricoteka/ui-core`

## Running Locally

Run the frontend and API together:

```sh
pnpm dev
```

Run only the frontend:

```sh
pnpm app-dev
```

The API runs on port `3000` by default. The frontend is served by Vite and uses hash-based routing.

## Common Commands

Run all package tests that exist:

```sh
pnpm test
```

Build the frontend:

```sh
pnpm app-build
```

Run frontend tests:

```sh
pnpm app-test
```

Typecheck the frontend:

```sh
pnpm app-typecheck
```

Lint the repository:

```sh
pnpm lint
```

Check formatting:

```sh
pnpm format
```

Apply formatting fixes:

```sh
pnpm format:fix
```

Run commands for a specific workspace package with pnpm filters:

```sh
pnpm --filter @pricoteka/app <script>
pnpm --filter @pricoteka/api <script>
pnpm --filter @pricoteka/core <script>
pnpm --filter @pricoteka/ui-core <script>
pnpm --filter @pricoteka/website <script>
```

Run only the website:

```sh
pnpm website-dev
```

## Frontend Development

The frontend entrypoint is `apps/frontend/src/main.tsx`.

Routes are file-based under `apps/frontend/src/routes/` and are generated by TanStack Router. Do not edit `apps/frontend/src/routeTree.gen.ts` directly.

Frontend scripts such as `dev`, `build`, `typecheck`, `test`, `lint`, and `format` run `prepare:checks` first. This validates i18n resources and regenerates the route tree.

For focused frontend tests, run the checks once and then run Vitest directly:

```sh
pnpm --filter @pricoteka/app prepare:checks
pnpm --filter @pricoteka/app exec vitest run <test-file>
```

Translations live in `apps/frontend/src/locales/<locale>/translation.json`. Non-default locales must match the default `en` message keys exactly.

Static assets in `apps/frontend/public/` are served from `/`. Frontend imports can use the `@/*` alias for `apps/frontend/src/*`.

Shared UI primitives are imported from per-component `@pricoteka/ui-core/*` exports, and the app imports shared theme CSS from `@pricoteka/ui-core/styles.css`.

## Website Development

The website entrypoint is `website/src/main.tsx`. It uses Vite, React, and the shared UI core package.

Common website commands:

```sh
pnpm website-dev
pnpm website-build
pnpm --filter @pricoteka/website lint
pnpm --filter @pricoteka/website format
```

## API Development

The API entrypoint is `apps/api/src/index.ts`. It is a Hono Node server with permissive CORS.

The database uses Drizzle with libSQL. `DB_FILE_NAME` defaults to `file:local.db`, and local database files under `apps/api/local.db*` are ignored by git.

Common API commands:

```sh
pnpm --filter @pricoteka/api dev
pnpm --filter @pricoteka/api build
pnpm --filter @pricoteka/api typecheck
pnpm --filter @pricoteka/api lint
pnpm --filter @pricoteka/api format
```

Database commands:

```sh
pnpm --filter @pricoteka/api db:generate
pnpm --filter @pricoteka/api db:migrate
pnpm --filter @pricoteka/api db:push
pnpm --filter @pricoteka/api db:studio
```

Edit database schema in `apps/api/src/db/schema.ts`, then regenerate migrations with Drizzle Kit. Do not hand-edit generated migration metadata.

## Core Package

The core package contains shared exports used by the frontend and API.

Its public surface is intentionally small:

- `packages/core/src/index.ts`
- `packages/core/src/schema.ts`

Common core commands:

```sh
pnpm --filter @pricoteka/core build
pnpm --filter @pricoteka/core test
pnpm --filter @pricoteka/core typecheck
pnpm --filter @pricoteka/core lint
pnpm --filter @pricoteka/core format
```

## UI Core Package

The UI core package contains shared shadcn UI primitives, shadcn utilities and hooks, and shared theme CSS used by the frontend and future website.

Use per-component exports from consumers:

```ts
import { Button } from "@pricoteka/ui-core/button";
```

Common UI core commands:

```sh
pnpm --filter @pricoteka/ui-core build
pnpm --filter @pricoteka/ui-core typecheck
pnpm --filter @pricoteka/ui-core lint
pnpm --filter @pricoteka/ui-core format
```

## Generated Files

Do not edit generated or build-output files directly:

- `apps/frontend/src/routeTree.gen.ts`
- `apps/frontend/build/dist/`
- `apps/frontend/dist/`
- frontend `stats*.html` and `stats.json`
- package `dist/` directories
- generated Drizzle metadata under `apps/api/drizzle/meta/`

## Verification Checklist

Before opening a pull request, run the checks relevant to the areas you changed.

For frontend changes:

```sh
pnpm --filter @pricoteka/ui-core build
pnpm app-build
pnpm app-test
```

For API changes:

```sh
pnpm --filter @pricoteka/api build
pnpm --filter @pricoteka/api lint
pnpm --filter @pricoteka/api format
```

For core changes:

```sh
pnpm --filter @pricoteka/core build
pnpm --filter @pricoteka/core test
pnpm --filter @pricoteka/core lint
pnpm --filter @pricoteka/core format
```

For UI core changes:

```sh
pnpm --filter @pricoteka/ui-core build
pnpm --filter @pricoteka/ui-core lint
pnpm --filter @pricoteka/ui-core format
pnpm app-build
pnpm app-test
```

For broad changes, also run:

```sh
pnpm lint
pnpm format
pnpm fallow:changed
```
